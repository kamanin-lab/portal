### useHostTokens.ts — canonical body (from 18-PATTERNS.md §9, 18-RESEARCH.md §Pattern 4 + §Security Domain)

```typescript
// Widget-side token handshake hook.
// Posts kmn/theme/request on mount, listens for kmn/theme/set, applies CSS vars,
// falls back to bundled defaults at 300ms if no reply.
//
// CRITICAL: return value MUST be useMemo-stabilized (feedback_react_hook_identity_churn + ADR-034).
// AppRenderer effect deps re-fire on every render if this hook returns a new object identity.

import { useEffect, useMemo, useRef, useState } from 'react'
import { WIDGET_TOKENS, DEFAULT_TOKEN_VALUES, type TokenKey } from '../widget-tokens'
import { PROTOCOL_VERSION, type ThemeSet } from '../types'

const FALLBACK_MS = 300

// Reject CSS values that don't match a safe token shape (hex, units, rgb(), etc.).
// Defends against postMessage-injected `expression(alert(1))` (IE-only, but defense-in-depth).
const SAFE_VALUE = /^[\d#.a-z%, ()/-]+$/i

type Source = 'host' | 'default'

export function useHostTokens() {
  const [tokens, setTokens] = useState<Record<TokenKey, string>>(DEFAULT_TOKEN_VALUES)
  const [source, setSource] = useState<Source>('default')
  const [protocolVersion] = useState<number>(PROTOCOL_VERSION)

  // Guards against the 300ms fallback overwriting a host reply that arrived first.
  const hostReplyReceivedRef = useRef(false)

  useEffect(() => {
    // 1. Send request immediately on mount.
    //    targetOrigin '*' because sandbox-proxy srcdoc has origin=null; identity gate on other side.
    window.parent.postMessage(
      { type: 'kmn/theme/request', protocolVersion: PROTOCOL_VERSION },
      '*',
    )

    // 2. Subscribe to replies (stays active for re-emits per MCPAPP-TOKEN-06).
    const onMessage = (e: MessageEvent) => {
      const data = e.data as ThemeSet | undefined
      if (!data || data.type !== 'kmn/theme/set') return
      if (typeof data.protocolVersion !== 'number') return

      // Protocol version asymmetry (MCPAPP-TOKEN-08).
      if (data.protocolVersion > PROTOCOL_VERSION) {
        console.warn(
          `[kmn-theme] host protocolVersion=${data.protocolVersion} > widget=${PROTOCOL_VERSION} — ignoring, staying on defaults`,
        )
        return
      }

      hostReplyReceivedRef.current = true

      // Apply tokens to CSS vars (MCPAPP-TOKEN-04).
      const applied: Record<TokenKey, string> = { ...DEFAULT_TOKEN_VALUES }
      for (const key of Object.keys(WIDGET_TOKENS) as TokenKey[]) {
        const value = data.tokens?.[key]
        if (typeof value === 'string' && SAFE_VALUE.test(value)) {
          applied[key] = value
          document.documentElement.style.setProperty(WIDGET_TOKENS[key], value)
        }
      }
      setTokens(applied)
      setSource('host')
    }
    window.addEventListener('message', onMessage)

    // 3. 300ms fallback (MCPAPP-TOKEN-05).
    const timerId = window.setTimeout(() => {
      if (hostReplyReceivedRef.current) return
      for (const key of Object.keys(WIDGET_TOKENS) as TokenKey[]) {
        document.documentElement.style.setProperty(WIDGET_TOKENS[key], DEFAULT_TOKEN_VALUES[key])
      }
      // `source` stays 'default'; `tokens` already = DEFAULT_TOKEN_VALUES via initial state.
    }, FALLBACK_MS)

    return () => {
      window.removeEventListener('message', onMessage)
      window.clearTimeout(timerId)
    }
  }, [])

  // useMemo-stabilized per ADR-034 + feedback_react_hook_identity_churn.
  return useMemo(() => ({ tokens, source, protocolVersion }), [tokens, source, protocolVersion])
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sandbox-proxy (parent frame) → widget iframe | Untrusted from widget's POV; sandbox-proxy is same-origin with portal, but widget must assume the postMessage payload could be crafted |
| Widget-inbound postMessage → `document.documentElement.style.setProperty` | CSS injection risk — a malicious `value` could theoretically include `url(javascript:...)` in a browser that allows it |
| `window.addEventListener('message')` → all incoming messages | Widget listens to ALL message events; must filter by `type === 'kmn/theme/set'` before acting |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-03-01 | Tampering / Elevation | Malicious postMessage with `tokens: { fg: 'url(javascript:...)' }` or `expression(alert(1))` | mitigate | `SAFE_VALUE = /^[\d#.a-z%, ()/-]+$/i` regex in `useHostTokens` MUST pass before `setProperty`. Values not matching are silently dropped (not logged — noise defense). `expression()` is IE-only but included for defense-in-depth. |
| T-18-03-02 | Spoofing | postMessage from an attacker-controlled same-origin iframe posing as sandbox-proxy | accept | Widget cannot validate `e.origin` because srcdoc has origin=null; widget trusts its parent frame identity. Trust is delegated to the portal's embed of `@mcp-ui/client` AppRenderer. This is a known limitation per 18-RESEARCH.md §Security Domain. |
| T-18-03-03 | Tampering | protocolVersion inflation attack (host sends protocolVersion: 999 to force widget into undefined behavior) | mitigate | Widget rejects `data.protocolVersion > PROTOCOL_VERSION` with console.warn + early return. Falls back to DEFAULT_TOKEN_VALUES at 300ms. MCPAPP-TOKEN-08 explicit contract. |
| T-18-03-04 | Denial of Service | StrictMode double-mount → two listeners, two timers, two requests | mitigate | `useRef` persists across effect invocations so `hostReplyReceivedRef` is deduplicated; the listener+timer are cleaned up on each unmount. Result: requests double in dev, but host response is idempotent (same tokens applied twice = same CSS state). Production StrictMode is off → no issue. |
| T-18-03-05 | Information Disclosure | Widget bundles reveal `DEFAULT_TOKEN_VALUES` hex colors | accept | Tokens are design-system values, NOT secrets. Leaking hex colors has zero security impact. Explicit in 18-RESEARCH.md §Security Domain "Tokens aren't secrets anyway". |
| T-18-03-06 | Tampering | Contract test drift — someone adds a 13th token only to mcp-poc side | mitigate | Vitest contract test fails on mcp-poc side whenever `Object.keys(WIDGET_TOKENS).sort() !== FROZEN_KEYS.sort()`. PORTAL-side has its own twin (Plan 04). Cross-repo CI blocks merge on either side's failure. |
