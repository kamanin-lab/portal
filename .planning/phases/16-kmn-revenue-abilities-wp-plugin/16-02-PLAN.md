<sql_corrections_from_research>
**CRITICAL** — these are the deltas from WP_BRIDGE_ARCHITECTURE.md that MUST be honored:

1. **`wc_order_stats` has NO `billing_email` column.** Any per-customer query MUST JOIN to `wc_orders`:
   ```sql
   SELECT o.billing_email, COUNT(*) AS c
   FROM   {prefix}wc_order_stats s
   JOIN   {prefix}wc_orders o ON s.order_id = o.id
   WHERE  s.date_created >= %s AND s.status IN (...)
   GROUP  BY o.billing_email
   ```

2. **`payment_method` is on `wc_orders` (top-level, denormalized).** NO `wc_orders_meta` lookup needed:
   ```sql
   SELECT o.payment_method, SUM(s.net_total)
   FROM   {prefix}wc_order_stats s
   JOIN   {prefix}wc_orders o ON s.order_id = o.id
   WHERE  s.date_created >= %s AND s.status IN (...)
   GROUP  BY o.payment_method
   ```

3. **Use numeric UTC offsets** (e.g. `'+01:00'`) in `CONVERT_TZ()`, NOT named tz (`'Europe/Vienna'`). The DDEV image happens to have tz tables populated, but production hosts may not. RESEARCH §C6 — portability decision.

4. **Always call `kmn_revenue_set_query_timeout_ms( 2000 )`** before the heavy SELECT in every ability (ABIL-QA-03).

5. **Use `wc_order_stats.date_created`** (not `date_created_gmt`) — both are populated identically in current WC and `date_created` is indexed. RESEARCH §C1.

6. **Never use `wc_order_stats.returning_customer`** — unreliable for guest checkouts. Phase 15 validation note + RESEARCH §C1.
</sql_corrections_from_research>

### 1.1 `abilities/weekly-heatmap.php`

Register `kmn/weekly-heatmap` using the skeleton above. Specifics:

**input_schema** (per WP_BRIDGE §4b):
- `weeks`: integer, `enum: [4,8,12,26,52]`, default 8
- `timezone`: string, optional (default wp_timezone())
- `reference_date`: string, `pattern: ^\d{4}-\d{2}-\d{2}$`, optional (default today in site tz)
- `status`: array of strings, default `['wc-completed','wc-processing']`

**output_schema:** per WP_BRIDGE §4b — `window_weeks`, `timezone`, `buckets` (array of {day_of_week 0-6, hour_of_day 0-23, order_count, net_revenue}), `best_slot` (day_of_week/hour_of_day/order_count), `calculated_at`.

**execute_callback:**
```php
global $wpdb;
kmn_revenue_set_query_timeout_ms( 2000 );
$weeks     = (int) ( $input['weeks'] ?? 8 );
$statuses  = kmn_revenue_status_whitelist( $input );
$offset    = kmn_revenue_resolve_tz_offset( $input );
$ref_date  = sanitize_text_field( $input['reference_date'] ?? current_time( 'Y-m-d' ) );
[ $start, $end ] = kmn_revenue_utc_bounds_for_window( $ref_date, $weeks * 7, $offset );
$placeholders = kmn_revenue_prepare_in_placeholders( $statuses );

$sql = $wpdb->prepare(
    "SELECT (DAYOFWEEK(CONVERT_TZ(s.date_created, '+00:00', %s)) - 1) AS dow,
            HOUR(CONVERT_TZ(s.date_created, '+00:00', %s))              AS hod,
            COUNT(*)              AS order_count,
            SUM(s.net_total)      AS net_revenue
     FROM   {$wpdb->prefix}wc_order_stats s
     WHERE  s.date_created >= %s AND s.date_created < %s
       AND  s.status IN ($placeholders)
     GROUP  BY dow, hod
     ORDER  BY dow, hod",
    array_merge( [ $offset, $offset, $start, $end ], $statuses )
);

$rows = $wpdb->get_results( $sql, ARRAY_A );
// Map MySQL DAYOFWEEK: 1=Sunday,2=Monday,...,7=Saturday → subtract 1 → 0=Sun...6=Sat
// WP_BRIDGE schema explicitly says 0=Sunday → matches.
$buckets = array_map( function ( $r ) {
    return [
        'day_of_week' => (int) $r['dow'],
        'hour_of_day' => (int) $r['hod'],
        'order_count' => (int) $r['order_count'],
        'net_revenue' => (float) $r['net_revenue'],
    ];
}, $rows );

// best_slot = max order_count; ties broken deterministically by (dow ASC, hod ASC)
$best = null;
foreach ( $buckets as $b ) {
    if ( ! $best || $b['order_count'] > $best['order_count']
       || ( $b['order_count'] === $best['order_count']
            && ( $b['day_of_week'] < $best['day_of_week']
                 || ( $b['day_of_week'] === $best['day_of_week'] && $b['hour_of_day'] < $best['hour_of_day'] ) ) ) ) {
        $best = $b;
    }
}

return kmn_revenue_response( true, [
    'window_weeks'  => $weeks,
    'timezone'      => $offset,
    'buckets'       => $buckets,
    'best_slot'     => $best ? [
        'day_of_week' => $best['day_of_week'],
        'hour_of_day' => $best['hour_of_day'],
        'order_count' => $best['order_count'],
    ] : null,
    'calculated_at' => gmdate( 'c' ),
] );
```

**Cache:** key via `kmn_revenue_cache_key('kmn/weekly-heatmap', $input)`, TTL `KMN_REVENUE_TTL_DEFAULT` (15 min).

### 1.2 `abilities/repeat-metrics.php`

Register `kmn/repeat-metrics`. Specifics:

**input_schema** (per WP_BRIDGE §4c):
- `days`: integer, `minimum: 30, maximum: 730, default: 90`
- `reference_date`: optional date
- `status`: array, default `['wc-completed','wc-processing']`

**output_schema:** per WP_BRIDGE §4c — `window_days`, `window_start`, `window_end`, `total_orders`, `unique_customers`, `new_customers`, `returning_customers`, `repeat_rate_pct`, `median_days_to_2nd` (nullable), `trend_pp`, `benchmark_pct: 27.0`, `calculated_at`.

**execute_callback — key structure:**
```php
global $wpdb;
kmn_revenue_set_query_timeout_ms( 2000 );
$days      = max( 30, min( 730, (int) ( $input['days'] ?? 90 ) ) );
$statuses  = kmn_revenue_status_whitelist( $input );
$offset    = kmn_revenue_resolve_tz_offset( $input );
$ref_date  = sanitize_text_field( $input['reference_date'] ?? current_time( 'Y-m-d' ) );
[ $start, $end ]         = kmn_revenue_utc_bounds_for_window( $ref_date, $days, $offset );
[ $prior_start, $prior_end ] = kmn_revenue_utc_bounds_for_window( $ref_date, $days * 2, $offset );
$prior_end = $start;  // prior window ends where current begins

$current = $this_scope_compute_repeat_rate( $start, $end, $statuses );
$previous = $this_scope_compute_repeat_rate( $prior_start, $prior_end, $statuses );
$trend_pp = round( $current['repeat_rate_pct'] - $previous['repeat_rate_pct'], 2 );

// median_days_to_2nd: ROW_NUMBER window over email-grouped orders
$placeholders = kmn_revenue_prepare_in_placeholders( $statuses );
$median_sql = $wpdb->prepare(
    "SELECT DATEDIFF(second_date, first_date) AS days_diff FROM (
         SELECT t.billing_email,
                MIN(CASE WHEN t.rn = 1 THEN t.date_created END) AS first_date,
                MIN(CASE WHEN t.rn = 2 THEN t.date_created END) AS second_date
         FROM (
             SELECT s.order_id, s.date_created, o.billing_email,
                    ROW_NUMBER() OVER (PARTITION BY o.billing_email ORDER BY s.date_created) AS rn
             FROM   {$wpdb->prefix}wc_order_stats s
             JOIN   {$wpdb->prefix}wc_orders o ON s.order_id = o.id
             WHERE  s.date_created >= %s AND s.date_created < %s
               AND  s.status IN ($placeholders)
         ) t
         GROUP BY t.billing_email
         HAVING second_date IS NOT NULL
     ) pairs",
    array_merge( [ $start, $end ], $statuses )
);
$diffs = $wpdb->get_col( $median_sql );
$median = null;
if ( count( $diffs ) > 0 ) {
    sort( $diffs, SORT_NUMERIC );
    $median = (float) $diffs[ intval( count( $diffs ) / 2 ) ];
}

return kmn_revenue_response( true, [
    'window_days'         => $days,
    'window_start'        => $start,
    'window_end'          => $end,
    'total_orders'        => $current['total_orders'],
    'unique_customers'    => $current['unique_customers'],
    'new_customers'       => $current['new_customers'],
    'returning_customers' => $current['returning_customers'],
    'repeat_rate_pct'     => $current['repeat_rate_pct'],
    'median_days_to_2nd'  => $median,
    'trend_pp'            => $trend_pp,
    'benchmark_pct'       => 27.0,
    'calculated_at'       => gmdate( 'c' ),
] );
```

Helper `compute_repeat_rate($start, $end, $statuses)` — **use closure inside execute_callback** (no namespace pollution):
```php
$compute = function ( $window_start, $window_end, $statuses ) use ( $wpdb ) {
    $placeholders = kmn_revenue_prepare_in_placeholders( $statuses );
    $sql = $wpdb->prepare(
        "SELECT o.billing_email, COUNT(*) AS cnt
         FROM   {$wpdb->prefix}wc_order_stats s
         JOIN   {$wpdb->prefix}wc_orders o ON s.order_id = o.id
         WHERE  s.date_created >= %s AND s.date_created < %s
           AND  s.status IN ($placeholders)
           AND  o.billing_email IS NOT NULL AND o.billing_email != ''
         GROUP  BY o.billing_email",
        array_merge( [ $window_start, $window_end ], $statuses )
    );
    $rows = $wpdb->get_results( $sql, ARRAY_A );
    $total = 0; $unique = 0; $returning = 0;
    foreach ( $rows as $r ) {
        $cnt = (int) $r['cnt'];
        $total += $cnt;
        $unique++;
        if ( $cnt >= 2 ) $returning++;
    }
    $new = $unique - $returning;
    $repeat_rate_pct = $unique > 0 ? round( $returning / $unique * 100, 2 ) : 0.0;
    return [
        'total_orders'        => $total,
        'unique_customers'    => $unique,
        'new_customers'       => $new,
        'returning_customers' => $returning,
        'repeat_rate_pct'     => $repeat_rate_pct,
    ];
};
```

### 2.1 `abilities/revenue-run-rate.php`

Register `kmn/revenue-run-rate`.

**input_schema** (per WP_BRIDGE §4a):
- `baseline_days`: integer, 7-56, default 14
- `timezone`: optional
- `reference_date`: optional
- `status`: array, default `['wc-completed','wc-processing']`

**output_schema:** `actual_now`, `current_hour`, `expected_by_hour[]` (24 elements), `projection` (nullable), `pace_vs_7day_pct`, `same_hour_last_week`, `payment_split` array of `{method, total, pct}`, `confidence` enum `[high,medium,low]`, `currency`, `calculated_at`.

**execute_callback** — three queries, then PHP branching:

```php
global $wpdb;
kmn_revenue_set_query_timeout_ms( 2000 );
$baseline_days = max( 7, min( 56, (int) ( $input['baseline_days'] ?? 14 ) ) );
$statuses  = kmn_revenue_status_whitelist( $input );
$offset    = kmn_revenue_resolve_tz_offset( $input );
$ref_date  = sanitize_text_field( $input['reference_date'] ?? current_time( 'Y-m-d' ) );
$placeholders = kmn_revenue_prepare_in_placeholders( $statuses );
$currency  = get_woocommerce_currency();

// 1. Baseline curve: per (date, hour) revenue over the baseline_days window
[ $base_start, $base_end ] = kmn_revenue_utc_bounds_for_window( $ref_date, $baseline_days, $offset );
$curve_sql = $wpdb->prepare(
    "SELECT DATE(CONVERT_TZ(s.date_created, '+00:00', %s)) AS d,
            HOUR(CONVERT_TZ(s.date_created, '+00:00', %s))  AS h,
            SUM(s.net_total) AS revenue
     FROM   {$wpdb->prefix}wc_order_stats s
     WHERE  s.date_created >= %s AND s.date_created < %s
       AND  s.status IN ($placeholders)
     GROUP  BY d, h
     ORDER  BY d, h",
    array_merge( [ $offset, $offset, $base_start, $base_end ], $statuses )
);
$curve_rows = $wpdb->get_results( $curve_sql, ARRAY_A );

// Build cumulative[d][h] via prefix sum within each day
$by_day = [];
foreach ( $curve_rows as $r ) {
    $by_day[ $r['d'] ][ (int) $r['h'] ] = (float) $r['revenue'];
}
$cumulative = [];
foreach ( $by_day as $day => $hours ) {
    $sum = 0.0;
    for ( $h = 0; $h <= 23; $h++ ) {
        $sum += $hours[ $h ] ?? 0.0;
        $cumulative[ $day ][ $h ] = $sum;
    }
}
// expected_by_hour: average over "valid" days where cumulative[d][23] >= 5.0
$expected_by_hour = array_fill( 0, 24, 0.0 );
$valid_days = [];
foreach ( $cumulative as $day => $hours ) {
    if ( ( $hours[23] ?? 0.0 ) >= 5.0 ) {
        $valid_days[] = $day;
        foreach ( $hours as $h => $v ) $expected_by_hour[ $h ] += $v;
    }
}
$valid_count = count( $valid_days );
if ( $valid_count > 0 ) {
    foreach ( $expected_by_hour as $h => $sum ) $expected_by_hour[ $h ] = round( $sum / $valid_count, 2 );
}

// 2. Today actual + current_hour
[ $today_start, $today_end ] = kmn_revenue_utc_bounds_for_date( $ref_date, $offset );
$actual_now_sql = $wpdb->prepare(
    "SELECT COALESCE(SUM(s.net_total), 0) AS total
     FROM   {$wpdb->prefix}wc_order_stats s
     WHERE  s.date_created >= %s AND s.date_created < %s
       AND  s.status IN ($placeholders)",
    array_merge( [ $today_start, $today_end ], $statuses )
);
$actual_now = (float) $wpdb->get_var( $actual_now_sql );

$now_in_tz = new DateTimeImmutable( 'now', new DateTimeZone( $offset ) );
$current_hour = ( $ref_date === $now_in_tz->format( 'Y-m-d' ) ) ? (int) $now_in_tz->format( 'H' ) : 23;

// 3. Payment split from wc_orders.payment_method (top-level column, per RESEARCH §C5)
$payment_sql = $wpdb->prepare(
    "SELECT COALESCE(NULLIF(o.payment_method, ''), 'other') AS method,
            SUM(s.net_total) AS total
     FROM   {$wpdb->prefix}wc_order_stats s
     JOIN   {$wpdb->prefix}wc_orders o ON s.order_id = o.id
     WHERE  s.date_created >= %s AND s.date_created < %s
       AND  s.status IN ($placeholders)
     GROUP  BY o.payment_method",
    array_merge( [ $today_start, $today_end ], $statuses )
);
$payment_rows = $wpdb->get_results( $payment_sql, ARRAY_A );
$payment_total = array_sum( array_map( fn( $r ) => (float) $r['total'], $payment_rows ) );
$payment_split = [];
foreach ( $payment_rows as $r ) {
    $total = (float) $r['total'];
    $payment_split[] = [
        'method' => (string) $r['method'],
        'total'  => $total,
        'pct'    => $payment_total > 0 ? round( $total / $payment_total * 100, 1 ) : 0.0,
    ];
}

// 4. Edge-case branching for projection + confidence
$projection = null;
$confidence = 'low';
if ( $valid_count < 5 ) {
    // sparse baseline
    $projection = null;
    $confidence = 'low';
} elseif ( $current_hour === 0 ) {
    // nothing to project from
    $projection = null;
    $confidence = 'low';
} elseif ( ( $expected_by_hour[ $current_hour ] ?? 0.0 ) < 5.0 ) {
    // too-small expected → fallback to 7-day ratio of full-day / same-hour
    // Load trailing 7 days subset
    $seven = array_slice( $valid_days, -7 );
    $sh_sum = 0.0; $eod_sum = 0.0; $n = 0;
    foreach ( $seven as $day ) {
        if ( ( $cumulative[ $day ][ $current_hour ] ?? 0.0 ) > 0 ) {
            $sh_sum  += $cumulative[ $day ][ $current_hour ];
            $eod_sum += $cumulative[ $day ][ 23 ];
            $n++;
        }
    }
    if ( $n > 0 && $sh_sum > 0 ) {
        $projection = round( $actual_now * ( $eod_sum / $sh_sum ), 2 );
        $confidence = 'medium';
    }
} else {
    // happy path
    $projection = round( $actual_now / $expected_by_hour[ $current_hour ] * $expected_by_hour[ 23 ], 2 );
    $confidence = 'high';
}

// 5. pace_vs_7day_pct: actual_now / avg(same-hour-last-7-days) - 1
$seven = array_slice( $valid_days, -7 );
$same_hour_last_week_vals = [];
foreach ( $seven as $day ) {
    $same_hour_last_week_vals[] = $cumulative[ $day ][ $current_hour ] ?? 0.0;
}
$same_hour_avg = count( $same_hour_last_week_vals ) > 0
    ? array_sum( $same_hour_last_week_vals ) / count( $same_hour_last_week_vals )
    : 0.0;
$pace_vs_7day_pct = $same_hour_avg > 0
    ? round( ( $actual_now / $same_hour_avg - 1 ) * 100, 1 )
    : 0.0;
// same_hour_last_week scalar: exact prior-week same day, same hour (if present)
$last_week_date = ( new DateTimeImmutable( $ref_date ) )->modify( '-7 days' )->format( 'Y-m-d' );
$same_hour_last_week = $cumulative[ $last_week_date ][ $current_hour ] ?? 0.0;

return kmn_revenue_response( true, [
    'actual_now'          => $actual_now,
    'current_hour'        => $current_hour,
    'expected_by_hour'    => array_values( $expected_by_hour ),  // 24-elem
    'projection'          => $projection,
    'pace_vs_7day_pct'    => $pace_vs_7day_pct,
    'same_hour_last_week' => (float) $same_hour_last_week,
    'payment_split'       => $payment_split,
    'confidence'          => $confidence,
    'currency'            => $currency,
    'calculated_at'       => gmdate( 'c' ),
] );
```

2. **AOV bands (always compute):** Parse `$aov_bands` as [b1, b2] (default [500, 1500]). Build three bands: `[0, b1]`, `[b1, b2]`, `[b2, ∞)`.
```php

sort( $bands_input );
$b1 = (float) $bands_input[0];
$b2 = (float) ( $bands_input[1] ?? $b1 * 3 );

    "SELECT SUM(CASE WHEN s.total_sales < %f THEN 1 ELSE 0 END)                         AS cnt_low,
            SUM(CASE WHEN s.total_sales >= %f AND s.total_sales < %f THEN 1 ELSE 0 END)  AS cnt_mid,
            SUM(CASE WHEN s.total_sales >= %f THEN 1 ELSE 0 END)                         AS cnt_high,
            SUM(CASE WHEN s.total_sales < %f THEN s.total_sales ELSE 0 END)              AS rev_low,
            SUM(CASE WHEN s.total_sales >= %f AND s.total_sales < %f THEN s.total_sales ELSE 0 END) AS rev_mid,
            SUM(CASE WHEN s.total_sales >= %f THEN s.total_sales ELSE 0 END)             AS rev_high,
            AVG(s.total_sales)                                                           AS avg_val,
            SUM(CASE WHEN 1=1 THEN s.total_sales ELSE 0 END)                             AS rev_total,
            COUNT(*)                                                                     AS order_total
     FROM   {$wpdb->prefix}wc_order_stats s
     WHERE  s.date_created >= %s AND s.date_created < %s
       AND  s.status IN ($placeholders)",
    array_merge( [ $b1, $b1, $b2, $b2, $b1, $b1, $b2, $b2, $start, $end ], $statuses )
);
// ... build $aov_bands array of 3 rows with label, min, max, order_count, order_pct, revenue_share ...
```

3. **Median order value:** small dataset (<1500 orders, 90d) — fetch `SELECT total_sales FROM ... ORDER BY total_sales`, compute median in PHP. Do NOT use window functions here — keep query simple.

4. **Basket pairs (only if mode != aov_bands):**

   **Product mode** — self-join `wc_order_product_lookup`:

   ```php
               COUNT(DISTINCT a.order_id) AS co_occ
        FROM   {$wpdb->prefix}wc_order_product_lookup a
   ```

   ```php
       array_column( $raw_pairs, 'pid_a' ),

       foreach ( $posts as $p ) $titles[ $p->ID ] = $p->post_title;
   ```
