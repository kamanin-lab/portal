3. At that point, insert the hoisted lookup:

```typescript
// ORG-BE-03: Hoisted nextcloud_client_root lookup (replaces 7 inline reads).
// Dual-read: org → profile → null. Individual actions check for null and return
// NEXTCLOUD_NOT_CONFIGURED / FORBIDDEN as they did before.
const org = await getOrgForUser(supabaseService, user.id);
const { data: profileRow } = await supabase
  .from("profiles")
  .select("nextcloud_client_root")
  .eq("id", user.id)
  .maybeSingle();
const clientRoot: string | null =
  org?.nextcloud_client_root ?? profileRow?.nextcloud_client_root ?? null;
```

Note: the fallback `profileRow` query uses the anon client `supabase` — RLS permits self-read on profiles. Do NOT use the service client for the profile query (keeps the fallback honest: if user cannot read their own profile, they shouldn't get nextcloud access).
