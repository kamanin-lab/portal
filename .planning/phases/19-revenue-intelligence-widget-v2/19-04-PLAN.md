### Sandbox-safe link pattern (source: v1 App.tsx lines 12-24)
```tsx
const AppContext = createContext<InstanceType<typeof App> | null>(null);
function AdminLink({ href, children }) {
  const app = useContext(AppContext);
  const onClick = (e) => {
    if (!app) return; // standalone dev harness
    e.preventDefault();
    void app.openLink({ url: href }).catch((err) => console.error("openLink failed:", err));
  };
  return <a href={href} onClick={onClick} rel="noopener noreferrer">{children}</a>;
}
```

### UI-SPEC HeuteBlock visual hierarchy (locked)
1. Block heading `Heute` + clock timestamp on the right
2. `Bisher` row: label Caption + value Body (`1.240 €`)
3. `Hochrechnung ▶` row: label Heading + value Display (`4.280 €`) + `bis 23:59` suffix Caption + `Bei aktuellem Tempo` caption
4. Pace row: `vs. Ø gleiche Stunde letzte 7 Tage` + signed percent in success/danger
5. Reference row: `Gleiche Stunde letzte Woche` + value
6. Payment-method bars (top revenue method at 100% opacity, 2nd at 60%, 3rd at 30%)
7. Divider
8. AttentionList sub-section (optional)

Confidence branches:
- `high` → step 3 renders bare `formatCurrency(projected_revenue)` in Display role
- `medium` → step 3 renders number + caption `(Schätzung, geringe Datenbasis)` in warning color
- `low` OR `h_now === 0` OR `baseline_days_used < 5` → step 3 REPLACES number with `Noch zu früh für Hochrechnung` (h_now===0) or `Nicht genug Daten für Hochrechnung` (insufficient baseline); step 2 (Bisher) still renders; pace row hidden
