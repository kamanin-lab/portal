## Issues Encountered

- **Standalone widget `app === null`:** When loaded outside an `@mcp-ui/client` host, the widget's `useContext(AppContext)` returns `null` (the dev-harness bypass branch). Period-toggle clicks become no-ops in this mode by design — they call `app?.callServerTool(...)` which short-circuits. This is correct behaviour for a fixture-only environment but means UAT-5 cannot be measured here at all. Logged as a deferred item, not a regression.

- **Vercel CDN measurement caveat for UAT-1:** First-paint of 496ms is impressively fast but is measured on Vercel's edge CDN serving a flat HTML+JS bundle, not a portal-embedded iframe with token publishing + AppBridge handshake + MCP App connect lifecycle. Real portal first-paint will include the full handshake chain. UAT-1 is recorded as PASS with the caveat that it represents a lower bound; the production-shape measurement remains worth taking inside HUMAN-UAT (added as informational note, not blocking).
