# Feature Idea: Система рекомендаций (Proactive Recommendations)

> Status: Idea | Priority: High | Target: Phase 4+

## Проблема

Агентство регулярно находит проблемы или возможности для улучшения у клиентов (SEO, производительность, безопасность, UX). Сейчас это сообщается устно или по email — нет системного способа предложить, оценить и получить решение клиента.

## Концепция

Агентство создаёт задачи-рекомендации в ClickUp с тегом **`рекомендация`**. Клиент видит их в портале как отдельную категорию с:

- Названием и описанием проблемы
- Предварительной оценкой в кредитах (→ см. `credit-system.md`)
- Действиями: **Принять** (выбрать дату) или **Отклонить**

## Поток

```
Агентство находит проблему / возможность
    │
    ▼ Создаёт задачу в ClickUp
    │   - Тег: "рекомендация"
    │   - Статус: специальный (напр. "RECOMMENDATION" или "VORSCHLAG")
    │   - Custom Field "Credits": предварительная оценка
    │   - Описание: что, почему, что будет если не сделать
    │
    ▼ Webhook → task_cache (тег + статус → тип "рекомендация")
    │
    ▼ Портал: клиент видит раздел "Empfehlungen" / "Vorschläge"
    │
    ├─► Клиент нажимает "Annehmen" (Принять)
    │       → Выбирает желаемую дату выполнения
    │       → Статус меняется на "TO DO"
    │       → Тег "рекомендация" остаётся (для аналитики)
    │       → Задача появляется в обычном списке задач
    │
    └─► Клиент нажимает "Ablehnen" (Отклонить)
        → Статус → "DECLINED" или помечается как отклонённая
        → Задача скрывается из активного списка
        → Остаётся в истории (можно вернуться позже)
```

## ClickUp: настройка

| Элемент | Значение |
|---------|----------|
| Тег | `recommendation` (или `empfehlung`) |
| Статус (новый) | `VORSCHLAG` — до решения клиента |
| Custom Fields | `Credits` (Number, Fibonacci), `Recommended Date` (Date, опционально) |
| Создаёт | Разработчик / Юрий / AI-агент (Phase 2+) |

## Portal UI

### Раздел "Empfehlungen"

Может быть:
- **Вариант A:** Отдельная вкладка/секция в модуле задач
- **Вариант B:** Фильтр в существующем списке задач (тег = рекомендация)
- **Вариант C:** Карточки в Dashboard / Overview

### Карточка рекомендации

```
┌─────────────────────────────────────────┐
│ 💡 SSL-Zertifikat läuft am 15.04 ab    │
│                                         │
│ Ihr SSL-Zertifikat muss vor dem         │
│ 15. April erneuert werden, sonst        │
│ zeigen Browser eine Sicherheits-        │
│ warnung an.                             │
│                                         │
│ ⚡ 2 Credits                            │
│                                         │
│ [Annehmen]  [Ablehnen]  [Details →]     │
└─────────────────────────────────────────┘
```

### Действия клиента

**Annehmen (Принять):**
1. Модальное окно с датапикером: "Bis wann soll das erledigt werden?"
2. Edge Function → ClickUp API:
   - Изменить статус: `VORSCHLAG` → `TO DO`
   - Установить due date
3. Задача появляется в обычном потоке

**Ablehnen (Отклонить):**
1. Опциональный комментарий: "Warum nicht?"
2. Edge Function → ClickUp API:
   - Изменить статус: `VORSCHLAG` → `DECLINED`
   - Добавить комментарий с причиной
3. Задача уходит в архив рекомендаций

## Фазы реализации

### Phase 1: Базовый поток

1. Настроить тег `recommendation` и статус `VORSCHLAG` в ClickUp
2. Обновить webhook для распознавания тега/статуса рекомендации
3. UI: секция "Empfehlungen" с карточками
4. Действия: Annehmen (с датой) / Ablehnen
5. Edge Functions: `accept-recommendation`, `decline-recommendation`

### Phase 2: AI-генерация рекомендаций

1. Автоматический аудит: AI анализирует сайт клиента периодически
2. Находит проблемы (SEO, скорость, безопасность, доступность)
3. Автоматически создаёт задачи-рекомендации в ClickUp
4. AI оценивает кредиты (→ интеграция с `credit-system.md` Phase 2)

### Phase 3: Аналитика

- Конверсия рекомендаций: сколько принято / отклонено
- Среднее время принятия решения
- Топ категорий рекомендаций
- Revenue impact: принятые кредиты за период

## Интеграция с кредитной системой

Рекомендации тесно связаны с кредитами (`docs/ideas/credit-system.md`):

- Каждая рекомендация имеет предварительную оценку в кредитах
- При принятии — кредиты резервируются из бюджета клиента (Phase 3 кредитов)
- Клиент видит стоимость до принятия решения
- AI-оценка кредитов работает и для рекомендаций (Phase 2 кредитов)

## Открытые вопросы

1. **Отдельный статус или тег?** `VORSCHLAG` как статус в ClickUp или фильтрация только по тегу?
2. **Уведомления:** Email/push при новой рекомендации?
3. **Срочность:** Некоторые рекомендации критичны (SSL expiry) — как выделить?
4. **Повторное предложение:** Можно ли предложить отклонённую рекомендацию повторно?
5. **Где показывать:** Отдельная секция vs. фильтр в задачах vs. dashboard виджет?
